页面置换算法

  1. 最优置换算法
    在最远将来才会被访问的页进行替换。
    但是。。。未来是无法预测的所以这是理想策略，
    别的算法都可以拿最优置换来进行比较的标准。

  2. FIFO 先进先出
    随机策略：和FIFO 一样都比较简单，通过随机来决定
    Belady现象：内存越大页数越多，别的算法页命中概率提高，FIFO，随机，反而命中概率降低。

    原因：置换特征和进程访问内存的动态特征是矛盾的。
        不符合栈算法的特点。 LRU就有栈算法的特点

  3. LRU Least-recently-userd 最近最少使用
    ·使用链表 ，替换时替换链表尾步，将新的放入链表头
    ·使用栈
    都需要查找替换，开销比较大
    但是程序局部性很差时 ，LRU会退化为FIFO
    MFU
    MRU
    LFU Least-Frequently-used最近最不经常使用

  4. 时钟页面置换
    页表项中有一个使用位 use bit ,当使用该页时硬件会把use bit 置1，所有页表被操作系统弄个一个环形列表有一个指针在某个位置，当要置换时观察指针所指的页的use bit 是否为0 如果为0 代表有一段时间没被使用过就将该页替换，否在去看下一个页知道找到响应页。
    每过一段时间，所有use bit 应该都会被置0 。
    这个算法虽然没有达到LRU 的最优情况，但是减少了开销，也足够简单。

脏页：该页是否已经被修改过数据，如果修改了写会硬盘开销就会比较大。
页表项一一般会加一个dirty bit 或 modified bit
clean page: 因为没有被修改过 硬盘中有相应的内容 可以直接释放就好了。

时钟页面置换时一般会替换 clean 干净的页面 并且 没被使用过的页。
11 01 00 找到第一个为00 的

以上都是局部页面置换算法,只是针对一个程序而言。
抖动问题

内存请求超额时，操作系统会不断的进行页面的置换，这种成为抖动 thrashing
